GEM v8.18.0 — Release Notes (current-state, replaces the full GEM_Changelog.txt in the runtime project)

CURRENT VERSION
  v8.18.0 product-completion release candidate (built on the v8.17.1 local release).

CURRENT BEHAVIOR
  • Canonical Final Decision Status (gem_final_status): every hand carries exactly ONE typed status —
    MISTAKE / CONDITIONAL / CLEARED / UNASSESSED ("Not reviewed") / UNGRADED ("No decision"). A
    gradeable hand with no positive/negative adjudication is UNASSESSED, never CLEARED; secondary
    reasons (suckout/flip/cooler/justified/read-dependent) never manufacture a grade. The hand card,
    sticky top bar and hand-list popup all read the one canonical status; zero contradictions.
  • PokerHandDisplay (gem_report_draft/_cards): one canonical card component (CardVM/HandVM,
    render_poker_hand, the poker-hand marker, accessible labels, unknown-card handling). The header,
    hand-list popup (fmtCardSpans) and Tournament Results exit-hand all render through it; same
    component on desktop and mobile.
  • Commentary Capsule: registers FACTUAL / COACHING / INSUFFICIENT_EVIDENCE; the zero-drop migration
    ledger balances on the real report (no commentary item silently dropped).
  • Villain Teaching: the seven-part lesson (what villain did / cue / read / confidence / safe exploit
    now / future exploit / guardrail) with a stable identity (tournament_id|player_hash), observation
    separated from inference, no-hindsight chronology, and a coverage inventory.
  • Tournament Results: one typed DataTable (gem_report_draft/_datatable) — exit hand as the final
    column via PokerHandDisplay, Top% to one decimal, muted-but-retained -100% ROI, signed Net/ROI,
    totals row + average Top%, sticky compact filters; Details/Drivers/SRC removed.
