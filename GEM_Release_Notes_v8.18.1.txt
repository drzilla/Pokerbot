1. WHAT THIS HOTFIX FIXES  (Villain Teaching "Next time …" future exploits)
--------------------------------------------------------------------------------

v8.18.0 shipped two accepted, non-blocking Villain Teaching defects. v8.18.1 fixes
both:

  (a) TRUNCATION — some future exploits were cut off mid-thought by a 40-word
      clamp on the canonical text. The clamp is REMOVED; the canonical future
      exploit is now stored as a COMPLETE sentence and the renderer shortens for
      display only (the full sentence is in the DOM, the title tooltip, and the
      accessibility layer). No ellipsis hides canonical content.

  (b) CUE MISMATCH — some future exploits were chosen from the broad villain
      ARCHETYPE rather than the specific observed cue, so they could teach a
      different tendency than the one observed. Generation is now CUE-FIRST:
        1. the specific cue family / line pattern (the structured signal wins;
           the cue text is the word-boundary-matched fallback),
        2. the current exploit and its action domain,
        3. the villain archetype ONLY as a last-resort fallback modifier.
      A donk-bet lesson now teaches how to play against the same donk-bet; a
      wide-calling lesson teaches isolating/value-betting; an over-bluffing line
      teaches widening call-downs — never a swapped tendency.

  Every eligible future exploit now carries typed provenance:
      cue_family · current_exploit_domain · future_exploit_source · alignment_reason
  future_exploit_source ∈ { CUE_TEMPLATE, CURRENT_EXPLOIT_TRANSFORM,
                            ARCHETYPE_FALLBACK, MANUAL_EXISTING }.

3. INDEPENDENT QUALITY GATE (new)
--------------------------------------------------------------------------------
  _qa_v8181_future_exploit_quality.py inspects the actual teaching payload and
  rendered report and asserts, for every eligible unique lesson: sentence
  completeness (terminal punctuation, no truncation marker / dangling clause /
  generic placeholder), cue alignment (addresses the same cue family, compatible
  action domain, no contradiction with the current exploit, no hindsight), and
  provenance. It runs seven mutation tests (remove last words; swap a donk future
  for an underbluffed-river future; swap wide-calling for sudden-aggression;
  generic placeholder; remove punctuation; null future; later-street/result
  justification) — each must be detected. The package builder ABORTS if any
  predicate is false.
