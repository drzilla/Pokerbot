v8.14.0-preview (2026-06-14) -- Slice D: Villain Exploitation v2
=========================================================================
Turns the villain teaching block from "visible" into "actionable". The per-hand
projection now answers a compact, fixed teaching contract and recommends a
candidate Natural8 client tag. No new detector, scoring, or read taxonomy — this
is still a pure projection of the already-stamped villain_intel; it invents

Per-hand teaching contract (gem_villain_teaching._teach_lines), in order:
  What villain did -> Cue -> Read -> Confidence -> Exploit now ->
  Exploit future -> Do not over-adjust -> Bounty -> Tag suggestion
(Renamed from the v8.13.0 labels: Villain/Now/Next time/Avoid over-adjusting.)

Natural8 candidate tags (gem_villain_teaching.suggest_natural8_tag): maps a
derived read family to Ron's client colour tag — Danger Reg (red, high-confidence
aggression), Solid Reg (purple), Nit/Rock (brown), Calling Station (orange,
sticky/loose-passive/station), Maniac/LAG (pink), Fish (blue), Whale (cyan), Fun
Rec/Gambler (lime). Weak evidence (low confidence, a single cue, hindsight-gated,
or an unmapped read) yields the explicit "Unsure / Tag-me-later" (yellow) tag —
a tag is never forced. The "Candidate" qualifier is dropped only at high
confidence; the read label itself reads "Candidate <X>" / "Unknown / Tag-me-later"
unless high confidence.

Evidence aggregation (gem_villain_teaching.build_villain_evidence_summary): a
deterministic per-villain roll-up grouped by STABLE villain id (not by alias —
aliases are display-only and truncated to 3 + "+N more"). Each row carries the
strongest object's read, confidence, evidence count, cue, exploit, and candidate
tag. Exposed as teaching_summary in build_villain_teaching() output; data only
(rendering of the summary table is a follow-up — not wired this slice).

labels, adds a Confidence line style, and renders the Tag suggestion line with a
small Natural8 colour swatch (data-tag-color, driven by the trailing "(colour)"
token) — shown even on a fallback read, since "Unsure / Tag-me-later" is itself
the takeaway. No modal/top-bar/street-header/queue change.

v8.13.0 (2026-06-14) -- Slice F: villain exploitation teaching layer

Turns raw villain evidence into coaching-grade exploit teaching. New module
gem_villain_teaching.py PROJECTS the already-stamped villain intelligence
(exploit_opportunities / evidence_atoms / read_states + the canonical coaching
maps) into one constrained "teaching object" per exploit / evidence note that
answers: villain_did, cue, archetype (+confidence +evidence_count), exploit_now,
future_exploit, do_not_overadjust, plus source_truth{evidence_atoms, decision_id,
no_hindsight}.

Hard guarantees (unit-tested, T-VT-01..14):
- INVENTS NOTHING. Every villain-fact field is copied verbatim from an existing
  stamped field (exp['evidence_text'] / ['suggests'] / ['so_what'] /
  ['recommended_exploit'] / ['exploit_read_label'] / SIGNAL_COACHING). Missing
  source -> field omitted; thin read -> a single fixed fallback line
  ("Read is weak - one cue only. Use as a review candidate, not an exploit
  mandate.").
- NO-HINDSIGHT is structural: showdown-leaked atoms (available_before_action_index
  = None) can never become a same-hand cue; their action copy is suppressed and
  source_truth.evidence_atoms carries only strictly-earlier hands.
- CONFIDENCE wired to real evidence: profiler/population reads cap at low; high
  needs >=8 atoms + >=2 same-family corroboration; a low read never forces a
  high-confidence mistake.
- do_not_overadjust is DERIVED guardrail copy keyed by confidence (never a
  villain-specific claim).
- PKO cover reuses the existing coverage_label string verbatim; no bounty BB/$
  is fabricated.
- Online vs live is a copy suffix + field only; a live read is never
  cross-applied into an online report.

Renderer: a compact, additive ".v25-teach" block in the modal context render
(archetype + confidence + evidence count, the over-adjust guardrail, PKO cover,
or the thin-read fallback). All copy is pre-built in Python so the renderer only
displays strings. No detector / dimension / archetype-taxonomy change; no report
redesign, no Hand Review Queue UI, no strategy-threshold change, no PKO math
change beyond display. (Browser render pending Chat-env QA - data layer is
unit-tested; gem_villain_intel return contract + all existing villain tests
unchanged.)

rev-2 (GPT product-fail fixes): (1) the modal teaching block now renders the
FULL pre-built teach_lines for non-fallback objects — Read / Villain (the
villain_did action) / Cue / Now / Next time / Avoid over-adjusting / Bounty —
